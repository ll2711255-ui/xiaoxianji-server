#### ❌ M2：订单详情响应解析错误

**文件**：`xiaoxianji-miniapp/src/pages/orders/detail/detail.vue:195`

```js
// 当前代码（错误）
const res = await get('/orders/' + orderNo.value)
const d = (res && res.data) || res || {}   // d = orderObject (DB row)
const o = d.order                           // undefined！服务端没有 .order 包装层

const res = await get('/orders/' + orderNo.value)
const o = (res && res.data) || res || {}    // o = orderObject (DB row)
```

**根因**：服务端 `GET /api/orders/:orderNo` 返回 `res.json({ data: order })`——订单对象直接在 `data` 中，而非 `data.order`。

### 详细说明

#### ❌ M4：非微信支付断头路

**完整分析**：

```
用户下单流程（支付宝/抖音小程序）：
  1. checkout.vue → POST /orders
       └── order.service.createOrder()
             ├── 计价 ✅
             ├── 锁库存 ✅
             ├── 写 MySQL ✅
             └── wxpay.jsapiPrepay() ❌ 只支持微信！返回 payment: null

  2. checkout.vue 收到 { orderNo, payment: null, payError: '...' }
       └── 显示错误弹窗："支付暂不可用，请在订单列表重试"

  3. 用户到订单详情 → 点击"去支付"
       └── POST /orders/:orderNo/pay
             └── wxpay.jsapiPrepay() ❌ 又是只支持微信！

  4. 正确的支付宝/抖音支付路径（前端从未调用）：
       ├── POST /api/payment/alipay-create  → 返回 orderStr
       │     └── 前端 my.tradePay({ orderStr })
       └── POST /api/payment/tt-create      → 返回 orderId + orderToken
             └── 前端 tt.pay({ orderId, orderToken })
```

**需要修复的位置**：

1. **`checkout.vue:doSubmit()`**：提交订单后，根据 `currentProvider` 选择正确的支付下单接口（而不是始终用 `POST /orders` 返回的微信 payment）
2. **`orders/detail/detail.vue:onRetryPayment()`**：重试支付时同样需要根据平台选择正确的支付下单接口
3. **`pay.js:callPay()`**：当前已支持三平台 provider 参数，只需要前端传入正确的支付参数即可

**结论**：⚠️ 通知机制完整，但状态展示依赖字段名转换层。

### 步骤 6：商家端执行发货

**状态机规则**（merchant.routes.js:13-21）：

```
paid ──accept──→ accepted ──weigh──→ weighed ──process──→ processing ──ready──→ ready ──deliver──→ delivering ──complete──→ completed
```

**"发货"对应 `deliver` 动作**：


|------|---------|---------|---------|
| 接单 | `POST .../accept` | `paid` | `accepted` |
| 称重 | `POST .../weigh` | `accepted` | `weighed` |
| **处理** | `POST .../process` | `weighed` | `processing` |
| **备货** | `POST .../ready` | `processing` | `ready` |
| 发货 | `POST .../deliver` | `ready` | `delivering` |
| 完成 | `POST .../complete` | `delivering`/`ready` | `completed` |

```json

```

**结论**：⚠️ 用户测试 spec 跳过了 `process` 和 `ready` 两个必要步骤。

### 修复建议

**方案 A：服务端统一转换**（推荐）

在 `server/src/config/db.js` 的 `query()` / `queryOne()` 中添加 camelCase 转换：

```js
// db.js — 新增
function toCamelCase(row) {
  if (!row || typeof row !== 'object') return row
  const mapped = {}
  for (const [key, value] of Object.entries(row)) {
    const camelKey = key.replace(/_([a-z])/g, (_, c) => c.toUpperCase())
    mapped[camelKey] = value
  }
  return mapped
}

async function query(sql, params = []) {
  const [rows] = await pool.execute(sql, params)
  return rows.map(toCamelCase)
}
```
