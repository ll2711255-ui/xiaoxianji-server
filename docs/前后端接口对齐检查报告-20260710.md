> **检查日期**：2026-07-10

### 3.1 A 类：幽灵调用（前端调用了不存在的接口）

#### A1 🔴 P0 — 小程序登录页调用不存在的支付宝/抖音手机号接口

- `POST /auth/alipay-phone` → [login.vue:195](xiaoxianji-miniapp/src/pages/login/login.vue#L195)
- `POST /auth/toutiao-phone` → [login.vue:198](xiaoxianji-miniapp/src/pages/login/login.vue#L198)
- 服务端仅有 `POST /auth/wx-phone`
- **影响**：支付宝/抖音用户无法绑定手机号 → 无法支付

#### A2 🔴 P0 — PC 端 Banner 管理路径错误

- `GET /banners` → [BannerManage.vue:65](小鲜鸡PC商家端/src/views/banners/BannerManage.vue#L65)
- `PUT /banners` → [BannerManage.vue:122](小鲜鸡PC商家端/src/views/banners/BannerManage.vue#L122)
- 正确路径：`GET/PUT /store/banners`（[store.routes.js:76,87](server/src/routes/store.routes.js#L76)）
- **影响**：Banner 管理功能完全不可用（404）

#### A3 🔴 P0 — PC 端 MobileStock 调用不存在的 `/merchant/products`

- `GET /merchant/products` → [MobileStock.vue:64](小鲜鸡PC商家端/src/views/mobile/MobileStock.vue#L64)
- 服务端 `/api/merchant` 下无此路由
- **影响**：手机端库存管理页面 404

#### A4 🔴 P0 — PC 端 MobileWeigh 调用不存在的 GET 接口

- `GET /merchant/orders/:no` → [MobileWeigh.vue:103](小鲜鸡PC商家端/src/views/mobile/MobileWeigh.vue#L103)
- 服务端商家操作是 `POST /api/merchant/orders/:orderNo/:action`，不存在 GET
- 正确路径应为 `GET /orders/:orderNo`

### 3.3 C 类：入参不匹配

#### C1 🔴 P0 — PC 端登录字段名 `username` → 应为 `phone`

[Login.vue:83-86](小鲜鸡PC商家端/src/views/Login.vue#L83-L86)：
```js
api.post('/auth/merchant-login', { username: form.username, password: form.password }, true)
```
服务端 [auth.routes.js:100](server/src/routes/auth.routes.js#L100)：
```js
const { phone, password } = req.body;
if (!phone || !password) return res.status(400).json(...)
```
→ `phone = undefined` → 永远返回 400 → **PC 端完全无法登录**

#### C2 🔴 P0 — 小程序 wx-phone 调用传参不一致

[login.vue:192](xiaoxianji-miniapp/src/pages/login/login.vue#L192)：`post('/auth/wx-phone', { code: e.detail.code })` → 传 `code`
[orders.vue:235](xiaoxianji-miniapp/src/pages/orders/orders.vue#L235)：`post('/auth/wx-phone', { phoneCode: e.detail.code })` → 传 `phoneCode`

服务端期望 `{ phoneCode, phone }` → login.vue 传 `code`，服务端收不到 → **首次登录绑定手机号失败**

#### C3 🟡 P1 — 小程序 login.vue 的 wx-phone 缺 skipAuth

[login.vue:192](xiaoxianji-miniapp/src/pages/login/login.vue#L192) 未传 `{ skipAuth: true }`。如果用户有旧 token 且已过期，request.js 的 401 自动刷新会触发 → 刷新失败 → `clearTokens()` 清掉刚拿到的 token。

#### C4 🟡 P1 — PC 端 OrderDetail 称重参数不匹配

[OrderDetail.vue:187-189](小鲜鸡PC商家端/src/views/orders/OrderDetail.vue#L187-L189) 传：
```js
{ actualWeight, pricePerJin, processingFee }
```
服务端 [merchant.routes.js:194](server/src/routes/merchant.routes.js#L194) 期望：
```js
{ actualWeight, weighPhoto, cardNumber }
```
→ 缺 `weighPhoto` 和 `cardNumber`；多传了 `pricePerJin` 和 `processingFee`（被忽略）

#### C5 🟡 P1 — PC 端 OrderDetail 退款参数不匹配

[OrderDetail.vue:199](小鲜鸡PC商家端/src/views/orders/OrderDetail.vue#L199) 传 `{}`（空对象）
服务端 [merchant.routes.js:225](server/src/routes/merchant.routes.js#L225) 期望 `{ actualWeight }`
→ `actualWeight` 为 `undefined`

#### C6 🟡 P1 — PC 端 MobileWeigh 称重参数结构完全不同

[MobileWeigh.vue:142](小鲜鸡PC商家端/src/views/mobile/MobileWeigh.vue#L142) 传：
```js
{ items: weightData }  // 数组结构
```
服务端期望：
```js
{ actualWeight, weighPhoto, cardNumber }  // 单值结构
```
→ 手机端称重提交完全失败

#### C7 🔵 P2 — PC 端 Dashboard 冗余参数 `type: 'online'`

[Dashboard.vue:120-121](小鲜鸡PC商家端/src/views/Dashboard.vue#L120-L121)：`type: 'online'`

服务端 [merchant.routes.js:37-38](server/src/routes/merchant.routes.js#L37-L38) 的 `type` 筛选对应 `order_info.type` 列（值为 `delivery`/`pickup`/`offline`），不存在 `'online'`。该参数被静默忽略。

### 3.4 D 类：返回字段不匹配

#### D1 🔴 P0 — 数据库列名 `status_label` → 全端用 `.status` 访问

`server/src/config/db.js:41-44` — `db.query()` 使用 `mysql2/promise` 的 `pool.execute()`，返回的行对象 key 就是 MySQL 原始列名（蛇形），**不做任何转换**。

证据：`store.routes.js` 中有显式的 `toCamelCase()` 函数 → 说明不存在全局驼峰转换层。

数据库列名是 `status_label`，但全部前端代码用 `.status` 取值 → **永远 `undefined`**。

影响面（每处都导致功能完全失效）：

| 端 | 文件 | 行号 | 症状 |
|---|---|---|---|
| 小程序 | orders.vue | 43 | `order.status === 'pending'` → "去支付"按钮永不出现 |
| 小程序 | orders.vue | 130 | `getStatusText(order.status, ...)` → 状态文本异常 |
| 小程序 | orders.vue | 144-151 | `getStatusClass(order.status)` → 标签颜色全灰 |
| 小程序 | orders.vue | 154-159 | `canCancel(order)` → 取消按钮逻辑全失效 |
| PC | Dashboard.vue | 109 | `getStatusText(o.status, o.type)` → 最近订单状态异常 |
| PC | OrderQueue.vue | 92-103 | `o.status === 'pending'` 等 → Tab 筛选失效 |
| PC | OrderQueue.vue | 132-158 | `getActions(order)` → 操作按钮全部不显示 |
| PC | OrderDetail.vue | 117-146 | `order.value.status` → 操作+称重区+步骤条全失效 |

#### D2 🔴 P0 — 数据库蛇形列名 → 全端驼峰取值

| 数据库列名 | 前端取值 | 影响 |
|---|---|---|
| `pay_amount` | `.prepayAmount` / `.payAmount` | 预付金额始终 0 / NaN |
| `actual_amount` | `.actualAmount` | 实付金额始终为空 |
| `refund_amount` | `.refundAmount` | 退款金额始终 0 |

### 4.5 错误码处理一致性检查

#### 服务端统一错误格式

```js
// 成功
{ success: true,  code: 200, data: {...}, message: '...' }
// 失败
{ success: false, code: NNN, message: '具体错误原因' }
```

HTTP 状态码使用：400（参数错误）/ 401（未登录/过期）/ 403（无权限）/ 404（不存在）/ 409（冲突）/ 500（服务器错误）

#### 错误处理对比

| | 小程序 | PC 端 |
|---|---|---|
| 非 2xx 处理 | 提取 `res.data.message` → `reject(new Error(msg))` ✅ | `Promise.reject(error)` → catch 中硬编码消息 ❌ |
| 401 处理 | 自动刷新 → 失败跳登录 ✅ | 自动刷新 → 失败跳登录 ✅ |
| 409 处理 | `catch(err)` → `err.message` 含服务器消息 ✅ | 硬编码 `'操作失败'` → 用户看不到 TOCTOU 提示 ❌ |
| HTTP 网络错误 | `'网络请求失败，请检查网络连接'` ✅ | axios 默认处理 |

#### ❌ 错误码问题

| # | 严重度 | 简述 |
|---|---|---|
| ERR1 | P1 | PC 端所有 catch 块丢弃服务端 `message` 字段，用户永远看不到真实错误原因 |
| ERR2 | P2 | PC 端 upload 401 刷新逻辑单独实现，与普通请求的刷新队列机制不同 |

## 七、技术背景（供修复参考）

### 7.1 HTTP 客户端行为摘要

**小程序 `request.js`**：
- 全路径 = `BASE_URL + '/api' + path`
- Token 存储 key：`access_token` / `refresh_token`
- `skipAuth: true` 时不带 Authorization 头（用于登录接口）
- 401 → `POST /api/auth/refresh-token` → 重试原请求（最多 1 次）
- upload 方法：`uni.uploadFile`，fieldname `'file'`

**PC 端 `api.js`**：
- baseURL = `API_BASE_URL + '/api'`
- Token 存储：Pinia `authStore.accessToken` / `authStore.refreshToken`
- `skipAuth: true` 时不带 Authorization 头
- 所有方法（get/post/put/del/patch）返回 `res.data`（已经 axios 解包）
- upload 方法走裸 `axios.post`，不走实例拦截器
