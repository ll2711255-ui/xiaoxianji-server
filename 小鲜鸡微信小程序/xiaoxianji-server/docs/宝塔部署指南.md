# 小鲜鸡后端服务 — 宝塔面板部署指南

## 一、服务器配置

| 项目 | 最低 | 推荐 |

## 四、部署后端代码

### 1. 创建目录并上传代码
```bash
# SSH 登录
mkdir -p /www/wwwroot/xiaoxianji/server/
# 将本地 xiaoxianji-server/ 目录上传到服务器（建议用宝塔文件管理器直接拖拽）
```

### 2. 安装依赖
```bash
cd /www/wwwroot/xiaoxianji/server/
npm install
npx prisma generate
```

### 3. 配置环境变量
```
宝塔 → Node.js 管理器 → 你的项目 → 环境变量

或直接创建 .env 文件：
```
```
cd /www/wwwroot/xiaoxianji/server/
cp .env.example .env
nano .env  # 编辑填入实际值
```

**必须填写的环境变量：**
```
DATABASE_URL="mysql://xiaoxianji:数据库密码@localhost:3306/xiaoxianji"
JWT_SECRET="生成一个64位随机字符串"
JWT_REFRESH_SECRET="生成另一个64位随机字符串"
WX_APPID="wx178d0ae59d576ce1"
WX_APP_SECRET="你的小程序AppSecret"
WXPAY_MCHID="你的微信支付商户号"
WXPAY_SERIAL_NO="你的API证书序列号"
WXPAY_APIv3_KEY="你的APIv3密钥"
WXPAY_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n你的私钥内容\n-----END PRIVATE KEY-----"
```

### 4. 启动服务
```bash
# 方式一：PM2 管理器（宝塔可视化）
宝塔 → PM2 管理器 → 添加项目
  ├── 启动文件：src/index.js
  ├── 项目目录：/www/wwwroot/xiaoxianji/server/
  ├── 项目名称：xiaoxianji-server
  └── 进程数：2

# 方式二：命令行
pm2 start ecosystem.config.js
pm2 save
pm2 startup  # 开机自启
```

## 五、配置 Nginx 反向代理

### 1. 添加站点
```
宝塔 → 网站 → 添加站点
  ├── 域名：api.你的域名.com
  ├── 根目录：/www/wwwroot/xiaoxianji/server/public/
  └── 勾选：创建FTP / 创建数据库（不勾）
```

### 2. 配置 SSL
```
宝塔 → 网站 → api.你的域名.com → SSL
  ├── Let's Encrypt → 申请证书
  └── 勾选：强制 HTTPS
```

### 3. 配置反向代理
```
宝塔 → 网站 → api.你的域名.com → 配置文件

# 在 server 块中添加：
location /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 60s;
}
location /uploads/ {
    alias /www/wwwroot/xiaoxianji/server/uploads/;
    expires 30d;
}
```

### 4. 验证配置
```bash
nginx -t          # 检查配置语法
nginx -s reload   # 重新加载
```

## 七、配置定时任务

```
宝塔 → 计划任务 → 添加

任务1：清理过期订单
  类型：Shell脚本
  周期：每5分钟
  脚本：
    curl -s http://127.0.0.1:3000/api/cron/clean-expired-orders > /dev/null

任务2：平台证书刷新
  类型：Shell脚本
  周期：每天 02:00
  脚本：
    curl -s http://127.0.0.1:3000/api/health > /dev/null
  （证书在支付回调中自动刷新，此任务仅保活）
```

## 九、故障排查

| 问题 | 排查方法 |
|------|---------|
| 502 Bad Gateway | `pm2 status` 检查进程是否存活 |
| 数据库连接失败 | `mysql -u xiaoxianji -p` 测试连接 |
| 支付回调不通 | 检查防火墙端口 443，Nginx 日志 `/www/wwwlogs/` |
| SSL 证书过期 | 宝塔 → SSL → 续签 |
| 内存不足 | 减少 PM2 实例数，或升级服务器 |
| 端口冲突 | `netstat -tlnp \| grep 3000` 检查端口 |
