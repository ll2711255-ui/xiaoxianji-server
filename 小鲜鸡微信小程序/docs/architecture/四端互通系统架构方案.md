# 小鲜鸡四端互通商用系统 — 架构设计方案

## 一、项目概述

从微信云开发（CloudBase）全量迁移至自建服务器架构，实现 **PC管理后台 + 用户小程序 + 线下收银端 + 独立后端服务** 四端互通，彻底规避云开发支付封禁风险，全部接入微信 APIv3 原生支付。

### 迁移范围

| 当前（云开发） | 迁移后（自建） |
|---|---|
| 49 个云函数，零散部署 | 统一 REST API 服务（~60 个接口） |
| `wx-server-sdk` + `cloud.database()` | Node.js/Express + Prisma ORM + MySQL |
| `cloud.getWXContext().OPENID` | JWT Bearer Token 认证 |
| `cloud.openapi.*` 云调用 | 直接调用微信官方 API + axios |
| 云存储 | 阿里云 OSS / 本地文件存储 |
| 商家功能内置小程序分包 | 独立 PC Web 管理后台（Vue3） |
| 收银内嵌商家订单页 | 独立收银 PC 端 / 保留收银小程序 |

### 复用资产

- 小程序所有页面 UI（`.wxml`/`.wxss`）可直接复用，仅替换数据层调用
- 业务逻辑（三段定价、订单状态机、称重退款、ESC/POS 打印指令）可完整迁移
- 支付 APIv3 集成代码（签名、验签、证书管理）已在 CloudBase 外独立实现，直接复用

### 核心表 DDL — orders

```sql
CREATE TABLE orders (
  id            BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  order_no      VARCHAR(32) NOT NULL UNIQUE,        -- 订单号 A25070200123
  user_id       BIGINT UNSIGNED,                     -- 用户ID（线下单可为NULL）
  merchant_id   BIGINT UNSIGNED NOT NULL,
  type          ENUM('delivery','pickup','offline') NOT NULL,
  status        ENUM('pending','paid','accepted','weighed','processing','ready','delivering','completed','cancelled') NOT NULL DEFAULT 'pending',
  
  -- 金额（单位：分）
  prepay_amount    INT UNSIGNED NOT NULL DEFAULT 0,  -- 预支付金额
  actual_amount    INT UNSIGNED,                     -- 实际金额（称重后）
  refund_amount    INT UNSIGNED,                     -- 退款金额
  refund_status    ENUM('none','processing','success','failed') DEFAULT 'none',
  
  -- 支付信息
  transaction_id   VARCHAR(64),                      -- 微信支付交易号
  pay_time         DATETIME,
  
  -- 配送信息
  delivery_address JSON,                             -- {name,phone,province,city,district,detail}
  delivery_time    DATETIME,
  delivery_fee     INT UNSIGNED DEFAULT 0,
  
  -- 履约信息
  pickup_time      DATETIME,                         -- 自提时间
  card_number      VARCHAR(8),                       -- 号码牌
  token            VARCHAR(32),                      -- 取货验证token
  
  -- 称重信息
  actual_weight    INT UNSIGNED,                     -- 实际重量（克）
  weigh_photo_url  VARCHAR(512),                     -- 称重照片
  
  -- 状态时间戳
  accept_time      DATETIME,
  complete_time    DATETIME,
  cancel_time      DATETIME,
  created_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at       DATETIME ON UPDATE CURRENT_TIMESTAMP,
  
  INDEX idx_user (user_id),
  INDEX idx_merchant (merchant_id),
  INDEX idx_status (status),
  INDEX idx_type (type),
  INDEX idx_created (created_at)
);
```


### 认证方案：JWT 双 Token

```
Access Token:  JWT (openid/merchantId + role + exp=2h)
                小程序存 wx.getStorageSync('token')
                PC后台存 localStorage

Refresh Token: 32位随机串，存 Redis，有效期 7 天
               过期后用 refresh token 换新 access token

商家登录:      phone + password → bcrypt验证 → 签发含 role=merchant 的JWT
小程序登录:    wx.login code → code2Session → 查/建用户 → 签发含 role=customer 的JWT
```

## 十、小程序改造要点

### 1. 新增 `utils/api.js`（替代 `cloud.callFunction`）

```javascript
const API_BASE = 'https://api.你的域名.com'

const request = async (method, path, data) => {
  const token = wx.getStorageSync('token')
  return new Promise((resolve, reject) => {
    wx.request({
      url: API_BASE + path,
      method,
      data,
      header: {
        'Authorization': token ? `Bearer ${token}` : '',
        'Content-Type': 'application/json'
      },
      success(res) {
        if (res.statusCode === 401) {
          // token过期，尝试刷新
          return refreshToken().then(() => request(method, path, data))
        }
        resolve(res.data)
      },
      fail: reject
    })
  })
}

// 使用方式：
// const res = await api.get('/products', { category_id: 'cat_01' })
// const res = await api.post('/orders', { items, address_id })
```

## 十二、风险与注意事项

| 风险 | 应对措施 |
|------|---------|
| 小程序切换 API 需重新审核 | 提前提交审核，保留旧版云开发作为回退 |
| 支付商户号可能需要重新申请 | 确认当前商户号支持直连模式 APIv3 JSAPI |

| 云数据库迁移数据丢失 | 提前全量导出 JSON，逐表校验行数 |
| 服务器性能不足 | 做压力测试，PM2 多实例 + Redis 缓存 |
| 微信支付证书过期 | 定时任务检测证书有效期，提前告警 |
